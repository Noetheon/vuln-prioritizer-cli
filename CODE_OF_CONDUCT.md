# Code of Conduct

`vuln-prioritizer` is maintained as a professional, security-focused open source project.

## Expected Behavior

Contributors and maintainers are expected to:

- communicate directly and respectfully
- keep feedback technical, specific, and actionable
- assume good intent while still challenging weak reasoning
- stay on-topic and avoid disruptive behavior
- protect private security details until maintainers agree they can be disclosed safely

## Unacceptable Behavior

The following behavior is not acceptable in project spaces:

- harassment, intimidation, or personal attacks
- discriminatory or demeaning language
- deliberate disruption of technical discussion
- publishing private credentials, sensitive data, or unreviewed exploit details
- bad-faith issue spam or abusive review behavior

## Maintainer Responsibilities

Maintainers may edit, remove, or reject comments, issues, pull requests, or other contributions that conflict with this code of conduct or the project's documented scope.

## Reporting

For conduct issues, contact the maintainer privately through the repository owner account. For security issues, follow [SECURITY.md](SECURITY.md).
